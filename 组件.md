BasePlayerInfoCachePool.cs — 玩家信息缓存池抽象基类
这是一个**通用的"按玩家ID缓存任意类型信息 + 异步请求合并 + LRU淘汰"**的抽象基类，把"前端缓存 + 服务器请求"这套套路统一封装好，子类只需要实现"如何向服务器发请求 / 如何收服务器消息 / 如何取自己的数据"即可。

UnionInfoCache.cs — 联盟（公会）信息缓存
继承自 BasePlayerInfoCachePool，把上面那一整套机制复用到"联盟信息"上。注意它把基类的"playerId"语义当作 unionID 来用——这是这个基类的常见复用方式：key 不一定是玩家 ID，凡是"按某个 ID 拉一个对象的数据"都能套。
