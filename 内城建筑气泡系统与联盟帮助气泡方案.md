# 内城建筑气泡系统分析 & 联盟帮助气泡开发方案

---

## 一、气泡系统架构

系统分两层：P2 通用框架提供底座，X11 在气泡 prefab 上直接挂 MonoBehaviour 组件实现业务。

```
P2 通用框架
  UIBuildingBubble        气泡管理器（对象池、优先级排序、UI 渲染）
  UIBuildingBubbleData    气泡数据（BubbleType、clickCallback、cfgID…）
  RBuilding.Bubble.cs     建筑端接口（AddBubbleData / RefreshBubble）

X11 业务气泡（MonoBehaviour，直接挂在 prefab 节点上）
  UIHospitalBubble        医院治疗气泡（可治疗 / 治疗中 / 可收取）
  UITrainSoldiersBubble   训练气泡
  UIBuildingBubbleResource 资源收集气泡
  UISurvivorBubble        幸存者派遣气泡（优先级最高）
  TechLabel               科技研究气泡（X11 Tech 模块）

优先级仲裁
  SurvivorBubbleBuildingPriority  幸存者气泡占用时压制同建筑其它气泡
```

### X11 气泡的统一实现模式

```csharp
// 1. 订阅 4 类事件
EventMgr.RegisterEvent(TEventType.XXX, OnXxx, this);               // 业务数据变化
EventMgr.RegisterEvent<BuildingType>(TEventType.ShowBuildingCircularMenu, OnShow..., this);
EventMgr.RegisterEvent<BuildingType>(TEventType.HideBuildingCircularMenu, OnHide..., this);
EventMgr1.RegisterEvent<BuildingType, bool>(
    TEventType1.SurvivorBubbleSuppressBuildingHeadBubblesChanged, OnSuppress..., this);

// 2. CheckToShow 统一判定
var canShow = 建筑存在 && buildingData.Level > 0
    && 业务条件
    && !SurvivorBubbleBuildingPriority.IsSuppressingBuildingBubbles(buildingType);
transform.localScale = canShow ? Vector3.one : Vector3.zero;   // 缩放显隐

// 3. HierarchySnapshot 切换多态
_snapshot.RestoreSnapshot("idle" / "progress" / "done");
```

## 二、联盟帮助气泡开发方案

### 2.1 目标

在以下三类建筑的头顶气泡中增加「请求联盟帮助」入口：

| 场景 | 建筑类型 | 触发条件 |
|---|---|---|
| 升级建筑 | 所有可升级建筑 | 建筑升级队列进行中，且未申请过帮助 |
| 研究科技 | ScienceMuseum | 科研队列进行中，且未申请过帮助 |
| 治疗伤病 | Clinic / Hospital | 治疗中，且未申请过帮助 |

帮助申请接口：`LAllianceHelp.I.SeekUnionHelpReq(queueId)`  
已帮助判定：`LAllianceHelp.I.IsHelped(queueId)`  
前提：玩家已加入联盟（`LPlayer.I.IsPlayerInUnion()`）

---

### 2.2 方案选择

**方案 A：在现有气泡上增加快捷帮助按钮（推荐）**

在已有气泡的「进行中（progress）」快照上增加一个「帮助」小按钮，点击直接发请求。适合升级建筑和治疗场景——已有气泡有 progress 态。

**方案 B：独立的联盟帮助气泡组件**

新建 `UIAllianceHelpBubble` MonoBehaviour，与其它气泡并列挂在 prefab 上，由 `SurvivorBubbleBuildingPriority` 控制优先级。适合科技场景（`TechLabel` 已独立，直接扩展即可）。

**本方案采用 A + B 混合：**
- 建筑升级 / 治疗 → 方案 A（改现有气泡的 progress 态）
- 科技 → 方案 B（直接扩展已有的 `TechLabel`）

---

### 2.3 新建 `UIAllianceHelpBubble`（通用基础组件）

三个场景共用同一套「帮助按钮」逻辑，抽出公共组件：

```csharp
// Assets/X11/Scripts/GameLogic/UI/Building/UIAllianceHelpBubble.cs
namespace X11.GameLogic
{
    /// <summary>
    /// 联盟帮助请求气泡按钮，挂在现有气泡 progress 节点下的帮助按钮 GameObject 上。
    /// </summary>
    public class UIAllianceHelpBubble : MonoBehaviour
    {
        [SerializeField] private Building.BuildingType buildingType;

        private long _queueId;

        public void SetQueueId(long queueId)
        {
            _queueId = queueId;
            Refresh();
        }

        private void OnEnable()
        {
            GetComponent<EventTriggerListener>()?.AddListener(EventTriggerType.Click, OnClick);
            EventMgr.RegisterEvent<long>(TEventType.BuildingUpdate, _ => Refresh(), this);
        }

        private void OnDisable()
        {
            GetComponent<EventTriggerListener>()?.RemoveListener(EventTriggerType.Click, OnClick);
            EventMgr.UnregisterEvent(this);
        }

        private void Refresh()
        {
            var canShow = _queueId > 0
                && LPlayer.I.IsPlayerInUnion()
                && !LAllianceHelp.I.IsHelped(_queueId);
            gameObject.SetActive(canShow);
        }

        private void OnClick(GameObject _, PointerEventData __)
        {
            if (_queueId <= 0 || !LPlayer.I.IsPlayerInUnion()) return;
            LAllianceHelp.I.SeekUnionHelpReq(_queueId);
            FloatTips.I.FloatMsg(LocalizationMgr.Get("LC_UNION_help_request_sent"));
            gameObject.SetActive(false);
        }
    }
}
```

### 2.6 科技研究帮助气泡

`TechLabel`（`Assets/X11/Scripts/GameLogic/Module/Tech/View/Behaviours/TechLabel.cs`）已是独立的科技头顶气泡组件，`RefreshViewContent` 中读取 `TechMgr.I.Data.GetPriorityQueue()` 获取当前研究队列。

在 `RefreshViewContent` 中增加帮助按钮刷新：

```csharp
private void RefreshViewContent()
{
    var (cityQueue, researching) = TechMgr.I.Data.GetPriorityQueue();
    techQueueItem.SetCfgId(researching?.researching ?? 0);

    // 新增：研究中且未帮助则显示帮助按钮
    var queueId = cityQueue?.ID ?? 0;
    _allianceHelpBubble?.SetQueueId(queueId);
}
```

同时需在 `TechLabel` prefab 上加 `Btn_alliance_help` 节点，`OnEnable` 中绑定。

### 2.7 需要确认的事项

1. **队列 ID 获取方式**
   - 建筑升级：需确认 `RBuilding.Bubble` 里获取进行中建造队列 ID 的具体字段
   - 治疗：需确认 `LHospital` 或类似管理器暴露治疗队列 ID 的接口
   - 科技：已有 `cityQueue.ID`，可直接用

2. **`IsHelped` 的时机**：`LAllianceHelp.I.IsHelped(queueId)` 依赖 `HaveSeekedUnionHelpNtf` 推送，需确认登录时是否已下发（当前 `PullHaveSeekedUnionHelpReq` 方法体为空）

3. **prefab 改动**：三个建筑 prefab 的气泡节点需美术/UI 配合增加帮助按钮子节点

4. **联盟帮助事件刷新**：`SeekUnionHelpAck` 返回后需刷新帮助按钮（当前 `LAllianceHelp` 只发事件 `AllianceSeekUnionHelpAck`，组件需监听此事件重新调用 `Refresh()`）

## 三、开发步骤建议

```
1. 确认三个队列 ID 的获取方式（读逻辑代码）
2. 新建 UIAllianceHelpBubble.cs
3. 在医院 prefab 的 progress 节点加帮助按钮，UIHospitalBubble 中接入
4. TechLabel 接入（改动最小）
5. 建筑升级气泡接入（需找到 RBuilding 侧升级气泡的具体实现文件）
6. 监听 AllianceSeekUnionHelpAck 刷新按钮状态
```
