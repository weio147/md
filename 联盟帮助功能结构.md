# D23 联盟帮助功能结构

## 一、功能概述

联盟帮助允许玩家对联盟成员的建造/治疗/科研队列发起帮助，缩短对方的队列时间。同时玩家可主动请求联盟帮助自己的队列。每次帮助可获得联盟贡献点（`vm:11151002`），每日有上限。

---

## 二、分层结构

```
UI 层（AllianceHelpList / UIAllianceNewHelp）
    ↓  调用
逻辑层 LAllianceHelp（单例）
    ↓  收发 protobuf
网络层（UnionHelpInfoReq/Ack、HelpUnionAllMemberReq/Ack、SeekUnionHelpReq/Ack）
```

---

## 三、逻辑层：`LAllianceHelp`
路径：`Assets/P2/GameScript/GameLogic/Alliance/LAllianceHelp.cs`

单例，负责所有网络收发与事件转发。

### 主要接口

| 方法 | 说明 |
|---|---|
| `UnionHelpInfoReq()` | 拉取当前联盟所有成员的帮助列表（`UnionHelpInfoAck`） |
| `HelpUnionAllMemberReq()` | 一键帮助所有可帮助的成员（`HelpUnionAllMemberAck`） |
| `SeekUnionHelpReq(queueId)` | 请求联盟帮助自己的指定队列 |
| `IsHelped(queueId)` | 判断某队列是否已申请过联盟帮助 |
| `GetDailyContributionMax()` | 从 `CX11ConstConfig.UnionHelpDailyMaxContribute` 读取每日帮助贡献上限 |
| `DailyContribution` | 当天已获得的帮助贡献值（`UnionHelpInfoAck.contribution`） |

### 监听的 protobuf 消息

| 消息 | 处理 | 转发事件 |
|---|---|---|
| `UnionHelpInfoAck` | 更新 `DailyContribution`，转发帮助列表 | `TEventType.AllianceHelpInfoAck` |
| `SeekUnionHelpAck` | 错误码非0则 FloatTips 提示 | `TEventType.AllianceSeekUnionHelpAck` |
| `HelpUnionAllMemberAck` | 错误码非0忽略 | `TEventType.AllianceHelpAllMemberAck` |
| `HaveSeekedUnionHelpNtf` | 更新已申请帮助的队列 ID 列表 | `TEventType.AllianceHaveSeekedUnionHelpNtf` |
| `BeUnionHelpedNtf` | 自己被帮助时通知 | `TEventType.AllianceBeUnionHelpedNtf` |

## 四、UI 层

### 4.1 帮助列表界面（两个版本，逻辑几乎相同）

| 类 | 路径 | 说明 |
|---|---|---|
| `AllianceHelpList` | `D23Code/UI/`（旧版入口） | 弹窗型（`PopUp`） |
| `UIAllianceNewHelp` | `UI/AllianceNew/` | 新版嵌入式，`assetPath` 为空（由父界面加载） |

两者逻辑完全一致，核心行为：

- `OnShown`：调用 `UnionHelpInfoReq()` 拉取数据，同时监听红点变化（红点数量变化时重新拉取）
- `OnAllianceHelpInfoAck`：
  1. 过滤掉已帮助且非自己的条目（已帮助的他人不展示）
  2. 排序：自己的请求排在最前
  3. 刷新列表、进度条、刷新倒计时
- `OnHelpBtnClick`：触发 `HelpUnionAllMemberReq()`，一键帮助全部
- 「帮助全部」按钮：所有他人条目都已帮助时隐藏/置灰
- 每秒更新刷新倒计时，倒计时归零后自动重新拉取
- 右上角「商店」按钮：跳转到联盟商店（`CStores.HordeShop`）

### 4.2 列表项数据结构（`UnionHelp` protobuf）

| 字段 | 含义 |
|---|---|
| `playerID` | 请求帮助的玩家 ID |
| `name` / `head` | 玩家名称与头像 |
| `typ` | 队列类型：`Build`=建造、`Heal`=治疗、`Research`=科研 |
| `cfgID` | 关联配置 ID（建造时为建筑 ID） |
| `toLevel` | 建造目标等级（0=拆除、1=新建、>1=升级） |
| `helpedTimes` / `maxTimes` | 已被帮助次数 / 最大可帮助次数（进度条数据） |
| `helpedTime` | 已缩短的时间（毫秒，用于 label 轮播显示） |
| `isHelped` | 当前玩家是否已帮助过该条目 |

### 4.3 帮助描述文本逻辑

`GetHelpDesc` 按 `typ` 分流：

- **Build**：区分拆除/新建/升级三种文案，读 `CBuilding` 配置取建筑名
  - `toLevel==0`：`LC_UNION_help_me_remove_parameter`
  - `toLevel==1`：`LC_UNION_help_me_build_parameter`
  - `toLevel>1`：`LC_BUILDING_request_help_build`（含等级）
- **Heal**：固定文案 `LC_UNION_help_me_repair_wounded_soldiers`
- **Research**：当前返回空字符串（逻辑已注释，待实现）

### 4.4 进度条 label 轮播

`FrameUpdateRegistry.RegisterLabelUpdate` 注册 `UpdateLab`，对**自己的帮助条目**交替显示两种文案：
- `label==1`：`已帮助次数/最大次数`
- `label==0`：已缩短时间的格式化字符串（渐隐动画，alpha 随 `pro` 变化）

## 五、请求帮助的入口

玩家可在以下场景主动请求联盟帮助自己的队列：

| 场景 | 入口 | 接口 |
|---|---|---|
| 建筑队列 | 建筑 UI 中「请求帮助」按钮 | `SeekUnionHelpReq(queueId)` |
| 医院治疗 | `D23UIHospital` 中帮助按钮 | `SeekUnionHelpReq(queueId)` |
| 科研队列 | `UIResearchLab`（已注释，待接入） | `SeekUnionHelpReq(queueId)` |
| 获取更多面板 | `GetMoreRequestHelpItemUI` | 通过聊天频道发求助（`SendActvSheepSeekHelpReq`）|

## 六、红点机制

- 红点路径：`RedPointPrimary.RedPointUnion` → `RedPointSecondary.RedPointUnionHelp`
- `AllianceHelpList/UIAllianceNewHelp.OnShown` 记录当前红点数，监听 `TEventType.RedPointUpdate`；红点数量变化时触发重新拉取帮助列表，保证实时性。

---

## 七、贡献上限配置

每日帮助贡献上限来自配置表：
```
CX11ConstConfig.I(CX11ConstConfig.UnionHelpDailyMaxContribute).Val
```
UI 中以进度条形式展示：`当日已获贡献 / 上限`。
